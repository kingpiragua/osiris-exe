# 13 — Production Pipeline

## Purpose

To define how a piece of content goes from idea to a recovered artifact in the
archive — the repeatable process and quality gates.

## Scope

Workflow stages (concept → script → art → assembly → integration → review →
release), roles, tools, asset standards, and definition-of-done. Not the creative
intent (Creative Direction) or the data model (Content System).

## Principles

- Repeatable and documented; every artifact follows the same path.
- Quality gates protect tone and the "What It Is NOT" boundaries.
- Verify before claiming done.

## Current Status

Template. The de facto pipeline today is ad hoc; this volume should formalize it
as cadence increases.

## Open Questions

- Toolchain for artwork and its CRT/degradation treatment.
- Review checklist that enforces Charter boundaries.
- Asset formats, sizes, and optimization standards.
