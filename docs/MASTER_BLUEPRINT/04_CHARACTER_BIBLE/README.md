# 04 — Character Bible

## Purpose

To hold canonical character definitions — identity, history, voice, and fixed
design rules — so every depiction stays consistent.

## Scope

The cast: biographies, relationships, motivations, voice/dialogue rules, and
non-negotiable visual design rules per character. Casting across artifact types.

## Principles

- Fixed design rules are absolute (e.g., a character's defining features must
  appear in every depiction).
- Voice is consistent across memories and artifacts.
- Characters serve theme and history, never the reverse.

## Current Status

Template. Character material exists in `osiris-motion-comic/canon/` and should be
migrated into per-character entries here.

## Open Questions

- Canonical naming and aliases across artifacts.
- Which characters are point-of-view anchors for recovered memories.
